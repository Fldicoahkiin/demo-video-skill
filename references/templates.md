# Structure templates

Write the Demo Flow before recording. Two built-in shapes; fill the blanks, then
derive scenes and narration from it.

## Hackathon submission (4–5 min)

Judges score "problem → product → evidence → future", not raw screen time.

| Time | Section | On screen | VO focus |
|---|---|---|---|
| 30–60s | Problem | landing / the painful status quo | who hurts, how they cope today, why it's bad |
| ~3 min | Product demo | the real product, step by step | the main flow, key features, inputs → results |
| 20–40s | Evidence | proof: on-chain data, hashes, JSON, tests, metrics | what you can verify, what you validated |
| 30–60s | Conclusion | summary / roadmap / repo | value now, the technical edge, what's next |

VO skeleton:

```
[Problem]   We built <project> for <user>. Today they struggle with <pain>,
            because <reason>. Our goal: make <outcome> possible.
[Demo]      Let me show you. First we <action> — the system <does what>. Here's
            the key result: <result>. Next, <second flow>; this matters because
            <value>. Behind the scenes, <technical proof>.
[Evidence]  Don't take our word for it — <verifiable proof on screen>.
[Conclusion] In short, <project> lets <user> <outcome>. The repo is public, the
            demo is live, and next we <future>.
```

## Product / launch short (60–90s)

```
0–10s   open on the product, name what it is in one line
10–30s  the one flow that matters, acted out (glide → act → hold on result)
30–55s  a second beat or the proof ("watch it verify", a live result)
55–75s  close on the value + where to get it
```

VO skeleton:

```
This is <product>. It <one-line value>.
Watch: <action> → <result the viewer can see>.
And <proof / second capability>.
<Product> — <who it's for>, <call to action>.
```

## Turning a flow into scenes

- One scene per beat in the flow. Name them `01-…`, `02-…` in order; `01-*`
  starts on a fresh visitor.
- Each scene's clip should be a touch longer than its narration — pad with a
  held frame.
- The proof beat (a verification, a live result, a chain explorer) is the
  differentiator: give it a hold and a slow zoom in the composition.
- Problem and Conclusion can be text scenes over the product's landing/summary —
  no separate slides needed.
